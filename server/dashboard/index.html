Quick Reference

This dashboard shows real-time status of your NAT Punchthrough server.

API Endpoints

GET  /api/health          — Server health check
GET  /api/games           — List active games
POST /api/games           — Register a new game
GET  /api/games/{id}      — Get game details
DEL  /api/games/{id}      — Remove a game (Auth: Bearer token)
POST /api/games/{id}/heartbeat — Keep game alive
GET  /api/games/{id}/turn — Get TURN credentials
POST /api/turn-credentials — Get TURN credentials (alt)
GET  /ws                  — WebSocket signaling
GET  /ws/signaling        — WebSocket signaling (alt)

Diagnostic Commands

# Check server health
curl http://localhost:8080/api/health

# Test STUN connectivity (requires stunclient)
stunclient your-server-ip

# View logs
docker compose logs -f server
docker compose logs -f coturn
